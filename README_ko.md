## 프로젝트 개요

일본어 면접을 준비하는 사람이 혼자서도 발음을 확인하고 개선할 수 있도록 만든
AI 기반 코칭 서비스입니다.

면접 영상이나 음성을 업로드하면 말하는 속도, 간격, 발음 명료도를 수치로 분석하고,
과거 연습 기록을 바탕으로 오늘 집중적으로 연습할 3분짜리 트레이닝 루틴을 제안합니다.

단순한 일회성 분석 툴이 아니라, 연습을 거듭할수록 더 정확한 피드백을 제공하는
**누적형 코칭 구조**로 설계했습니다.

---

## 주요 기능

### 레퍼런스 모드
입력한 문장을 녹음하면, 원문과 실제 발화를 단어 단위로 비교합니다.
오류·누락·추가된 단어를 시각적으로 표시하고 심각도(상 / 중 / 하)를 제시합니다.

### 자유 발화 모드
정해진 문장 없이도 자유롭게 말한 내용을 분석합니다.
말하는 속도(WPM), 포즈 횟수, Whisper 인식 신뢰도를 바탕으로 발음 품질 점수(0~100)를 산출합니다.

### 3분 트레이닝 루틴
분석 결과를 바탕으로, 그날 수정해야 할 부분을 집중적으로 연습하는 3단계 드릴을 제시합니다.
실제로 틀린 단어를 반복 연습하는 구조라 일반적인 예문 추천보다 효과적입니다.

### 누적 코칭
로그인 유저의 연습 기록을 누적하여, AI가 최근 5회의 기록을 참조해
"저번보다 필러가 줄었네요" 같은 맥락 있는 피드백을 제공합니다.

## 시스템 구성

```
음성 입력 (녹음 또는 파일 업로드)
        ↓
  [FFmpeg]  동영상 → 음성 추출 · 형식 변환
        ↓
  [Whisper]  음성 → 텍스트 + 단어 타임스탬프 + 신뢰도 점수
        ↓
  [Transcript Analyzer]  WPM · 포즈 횟수 · 명료도를 알고리즘으로 수치화
        ↓
  [Diff 엔진]  원문 vs 인식 결과를 단어 단위로 비교 (레퍼런스 모드)
        ↓
  [GPT-4o-mini]  수치 데이터를 바탕으로 코칭 피드백 · 루틴 생성
        ↓
  [MongoDB]  세션 저장 → 누적 피드백에 활용
```

## 기술적 과제와 해결

**Problem**

같은 녹음 데이터를 보내도 다른 피드백이 생성되는 문제가 발생했습니다.
LLM이 주관적으로 평가하는 구조였기 때문에 평가 기준이 불안정하고,
유저가 실제 성장을 확인하기 어려운 상태였습니다.

**Solution**

유저가 "읽어야 할 문장"을 직접 입력할 수 있는 기능을 도입해 비교 기준이 되는 정답 데이터를 고정했습니다.
평가는 알고리즘(거리 계산)이 담당하고, LLM은 분석 결과를 유저가 이해할 수 있는 언어로 변환하는 역할만 담당하도록 분리했습니다.

**Result**

같은 녹음 데이터에 대해 항상 같은 비교 결과를 얻을 수 있게 되어 평가의 일관성을 확보했습니다.
피드백도 항상 실제 분석 데이터를 기반으로 생성되게 되었습니다.

---

## 설계에서 고민한 점

**LLM이 근거 없이 평가하지 않도록 평가 구조를 설계했습니다.**

LLM에게 "발음을 평가해줘"라고 맡기면, 실제 음성을 직접 평가할 수 없고
텍스트 정보만을 기반으로 판단하기 때문에 할루시네이션이 발생할 가능성이 있었습니다.

그래서 LLM이 개입하기 전에 Whisper 데이터를 기반으로
말하는 속도·포즈 횟수·인식 신뢰도를 수치로 측정하고, 그 수치를 평가 근거로 넘기는 구조로 만들었습니다.
**LLM은 평가를 하는 게 아니라, 이미 측정된 결과를 해석하는 역할만 담당합니다.**

## 실행 방법

### 필요한 것
- Docker / Docker Compose
- OpenAI API 키

### 셋업

```bash
git clone <repository-url>
cd jp-interview-coach

# 환경 변수 설정
cp backend/.env.example backend/.env
# OPENAI_API_KEY 입력

# 실행
docker-compose up --build
```

앱: http://localhost

> ※ OpenAI API 이용 비용을 고려해 공개 배포는 하지 않습니다.
